# 00 — نظرة عامة على نظام التثبيت

---

## ما هو نظام التثبيت؟

نظام التثبيت (Web Installer) هو آلية نشر آمنة ذات استخدام واحد، مُدمجة داخل مشروع Laravel. تتيح هذه الآلية تثبيت التطبيق على بيئة الإنتاج عبر المتصفح مباشرةً، دون الحاجة إلى الوصول عبر SSH أو تشغيل أوامر يدوية في الطرفية.

عند فتح رابط التثبيت لأول مرة، يُعرض على المستخدم نموذج يطلب بيانات الإعداد الأساسية (اسم التطبيق، عنوان URL، بيانات قاعدة البيانات)، ثم يُنفَّذ الإعداد الكامل تلقائياً.

---

## لماذا يوجد هذا النظام؟

في بيئات الاستضافة المشتركة (Shared Hosting)، لا يتوفر عادةً وصول SSH مباشر. لذلك لا يمكن تشغيل أوامر مثل:

```bash
php artisan migrate
php artisan key:generate
```

يحلّ نظام التثبيت هذه المشكلة بتنفيذ جميع عمليات الإعداد من داخل Laravel نفسه عبر `Artisan::call()` (واجهة PHP البرمجية لـ Artisan)، دون الاستعانة بـ `shell_exec` أو `exec`.

### المشاكل التي يحلّها:

| المشكلة | الحل |
|---|---|
| لا يوجد SSH على الاستضافة المشتركة | واجهة ويب تستدعي Artisan داخلياً |
| الحاجة إلى ضبط `.env` يدوياً | النظام يكتب `.env` تلقائياً من `.env.example` |
| خطر تكرار التثبيت | ملف `installed.lock` يمنع أي إعادة تشغيل |
| إدارة APP_KEY | يتم توليدها تلقائياً عبر `key:generate` |
| المايجريشن والسيدرز | تُنفَّذ تلقائياً بعد التحقق من الاتصال بقاعدة البيانات |

---

## متى يُستخدَم هذا النظام؟

يُستخدَم نظام التثبيت **مرة واحدة فقط** عند:

- **أول نشر للمشروع** على سيرفر الإنتاج أو الاستضافة.
- **عند تسليم المشروع** لعميل أو فريق تقني لا يملك معرفة بأوامر Laravel.
- **عند النشر على استضافة مشتركة** لا تتيح الوصول عبر SSH.

> ⚠️ بعد اكتمال التثبيت، يتعطل النظام تلقائياً ولا يمكن تشغيله مجدداً.

---

## الفرق بين بيئة التطوير وبيئة الإنتاج

| العنصر | التطوير المحلي | الإنتاج (Production) |
|---|---|---|
| ملف الإعداد | `.env` موجود ومهيَّأ يدوياً | `.env` يُنشئه المثبِّت تلقائياً |
| `APP_ENV` | `local` | `production` |
| `APP_DEBUG` | `true` | `false` (إجباري) |
| قاعدة البيانات | SQLite أو MySQL محلي | MySQL على السيرفر |
| تشغيل Artisan | مباشرة من الطرفية | عبر `Artisan::call()` داخل المثبِّت |
| `installed.lock` | غير موجود | يُنشأ بعد التثبيت |
| `install.php` | غير مطلوب | يُستخدَم مرة واحدة ثم يُحذف |
| `LOG_LEVEL` | `debug` | `error` |

## ملخص تدفق التثبيت الكامل

```
[المطور محلياً]
      │
      ▼
تشغيل: php generate-installer-token.php
      │  → يُولِّد توكن عشوائي (64 حرف hex)
      │  → يحفظه في: storage/app/installer-token.txt
      │
      ▼
رفع ملفات المشروع على السيرفر
(بما فيها installer-token.txt)
      │
      ▼
توجيه الدومين إلى مجلد /public
      │
      ▼
إنشاء قاعدة بيانات MySQL يدوياً من لوحة التحكم
      │
      ▼
فتح المتصفح → https://yourdomain.com/install.php
      │  → يتحقق من عدم وجود installed.lock
      │  → يقرأ التوكن من installer-token.txt
      │  → يُعيد التوجيه إلى /installer/{token}
      │
      ▼
عرض نموذج الإعداد (setup.blade.php)
      │  → اسم التطبيق، رابط URL، بيانات قاعدة البيانات
      │
      ▼
[POST] إرسال النموذج → InstallerController::runInstall()
      │
      ├── التحقق من البيانات (Validation)
      ├── اختبار الاتصال بقاعدة البيانات (PDO)
      ├── كتابة ملف .env
      ├── config:clear
      ├── key:generate --force
      ├── migrate --force
      ├── db:seed --force
      ├── storage:link (غير حرجة)
      └── optimize
      │
      ▼
إنشاء: storage/app/installed.lock
حذف:  storage/app/installer-token.txt
حذف:  public/install.php (إن أمكن)
      │
      ▼
عرض صفحة النجاح (success.blade.php)
      │
      ▼
[المثبِّت معطَّل نهائياً]
```
